# Credit Card Approval Prediction System

A complete, hands-on machine learning project that predicts whether a credit
card application will be **approved** or **rejected**, using Python, Flask,
Scikit-Learn/XGBoost, and (optionally) IBM Watson Machine Learning for cloud
deployment.

## 2. About the Dataset

This project ships with `data/generate_data.py`, which generates a realistic
**synthetic** dataset (8,000+ rows) with the same structure as the well-known
Kaggle "Credit Card Approval Prediction" dataset вЂ” including missing values
and duplicate rows on purpose, so the preprocessing step has real work to do.

Columns: `Gender, Own_Car, Own_Realty, Children_Count, Annual_Income,
Income_Type, Education_Level, Family_Status, Housing_Type, Age_Years,
Employment_Years, Has_Work_Phone, Has_Phone, Has_Email, Family_Members,
Overdue_Months, Target_Reject`.

**If you want to use the real dataset instead** (recommended for an actual
submission/portfolio piece), download it from Kaggle:
https://www.kaggle.com/datasets/rikdifos/credit-card-approval-prediction
(`application_record.csv` + `credit_record.csv`), merge them on `ID`, and
convert the multi-class `STATUS` column into a binary `Target_Reject` label
(e.g. `STATUS` of `2,3,4,5` в†’ high risk/rejected, else approved). Save the
result as `data/credit_card_approval.csv` with the same column names above,
and everything downstream works unchanged.

## 3. Setup

### 3.1 Requirements
- Python 3.8+
- (Recommended) Anaconda Navigator + Jupyter Notebook for the EDA step
- VS Code or PyCharm for the Flask app

### 3.2 Install dependencies
```bash
cd credit_card_approval
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

> **Note on XGBoost:** if `pip install xgboost` fails on your machine (rare,
> usually a platform issue), `models/train_model.py` automatically falls
> back to `GradientBoostingClassifier` so the pipeline still runs end-to-end.
> For your actual project write-up, make sure xgboost installs correctly вЂ”
> it's one of the required skills.

## 4. Run the Project Step by Step

### Step 1 вЂ” Generate the dataset
```bash
python data/generate_data.py
```

### Step 2 вЂ” Exploratory Data Analysis
```bash
python notebooks/01_eda.py
```
This saves 6 charts into `plots/`: target class balance, income type vs
approval, education vs approval, income distribution, age distribution, and
a correlation heatmap. (For your actual coursework submission, copy the code
in this file into a real Jupyter Notebook, since the assignment explicitly
requires one.)

### Step 3 вЂ” Train all 4 models
```bash
python models/train_model.py
```
This handles missing values, removes duplicates, label-encodes categorical
columns, trains **Logistic Regression, Decision Tree, Random Forest, and
XGBoost**, prints accuracy/confusion matrix/classification report for each,
and saves the best one to `models/best_model.pkl`.

Example result on the synthetic dataset:

| Model | Accuracy |
|---|---|
| Logistic Regression | ~85.4% |
| Decision Tree | ~82.2% |
| Random Forest | **~85.9% (best)** |
| XGBoost | ~85вЂ“87%* |

\* varies slightly run to run; XGBoost and Random Forest are typically the
top two performers on this kind of tabular data.

### Step 4 вЂ” Run the Flask web app
```bash
python app.py
```
Open **http://127.0.0.1:5000** in your browser. Fill in the applicant's
details and click "Predict Approval" to see an instant Approved/Rejected
result with the model's confidence score.

## 5. (Optional) Deploy to IBM Watson Machine Learning

This makes the model callable as a scalable, real-time REST API from the
cloud, per the project's original scope.

1. Create a free [IBM Cloud](https://cloud.ibm.com) account.
2. Create a **Watson Machine Learning** service instance from the IBM Cloud catalog.
3. Generate an API key: *Manage в†’ Access (IAM) в†’ API keys в†’ Create*.
4. Create a **Deployment Space** in Watson Studio and copy its Space ID.
5. Open `deploy_watson.py` and fill in `WML_API_KEY`, `WML_URL`, `SPACE_ID`.
6. Run:
   ```bash
   pip install ibm-watson-machine-learning
   python deploy_watson.py
   ```
7. The script prints a **scoring endpoint URL** вЂ” you can now call your model
   from anywhere (curl, Postman, another app) instead of only locally.

## 8. Troubleshooting

- **`ModuleNotFoundError: No module named 'xgboost'`** в†’ `pip install xgboost`, or rely on the automatic GradientBoosting fallback.
- **Flask app shows "model not found"** в†’ run `models/train_model.py` first; `app.py` loads `models/best_model.pkl` at startup.
- **Predictions look wrong for a category value** в†’ the dropdown values in `index.html` are generated directly from `models/encoders.pkl`, so as long as you re-run `train_model.py` after any data changes, they'll stay in sync.
